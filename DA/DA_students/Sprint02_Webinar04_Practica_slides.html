Metas de contenido para esta sesión

Al finalizar la sesión podrás:

  1. Duplicar datos crudos y trabajar sobre una hoja limpia sin destruir la fuente.
  2. Detectar problemas de calidad: fechas como texto, IDs con espacios, ventas como texto, booleanos no estandarizados, duplicados y llaves sin match.
  3. Crear columnas limpias para store_id, dept, week_date, weekly_sales e is_holiday.
  4. Enriquecer ventas con datos de tienda y departamento usando XLOOKUP o VLOOKUP.
  5. Construir KPIs de eficiencia, crecimiento y participación.
  6. Crear un dashboard mínimo con tablas dinámicas, gráficos y formato condicional.
  7. Documentar supuestos, validaciones y cambios en hojas auxiliares.
  8. Comunicar hallazgos usando Cifra → Factor → Iniciativa.

Parte 1 — Limpieza de datos (16 min)

1.1 Duplicar hoja de trabajo

Duplica raw_sales y renómbrala:

clean_sales

No modifiques raw_sales.

1.2 Renombrar columnas

Estandariza encabezados en clean_sales:

Original Nuevo
Store store_id_raw
Dept dept_raw
Date week_date_raw
Weekly Sales weekly_sales_raw
IsHoliday is_holiday_raw

1.3 Crear columnas limpias

Agrega estas columnas al final:

Columna nueva Propósito
store_id tienda convertida a número
dept departamento convertido a número
week_date fecha semanal estandarizada
weekly_sales venta semanal numérica
is_holiday indicador normalizado
week_key llave temporal año-semana

Fórmulas guía

Para quitar espacios:

=TRIM(A2)

Para convertir a número:

=VALUE(TRIM(A2))

Para crear semana:

=YEAR(C2)&"-"&TEXT(WEEKNUM(C2,2),"00")

Para normalizar feriado, define una regla sencilla:

Valor original Valor limpio
TRUE, YES, 1 1
FALSE, NO, 0 0

Parte 2 — Enriquecimiento con dimensiones (15 min)

El archivo de ventas solo tiene IDs. Para analizar correctamente se deben traer atributos desde dimensiones.

2.1 Crear columnas enriquecidas

En clean_sales, agrega:

Columna Fuente
store_name raw_stores
store_type raw_stores
size_m2 raw_stores
city raw_stores
dept_name raw_departments
dept_group raw_departments

2.2 Fórmulas sugeridas

Con XLOOKUP:

=XLOOKUP([@store_id], raw_stores!A:A, raw_stores!B:B, "No Match")

Con VLOOKUP:

=VLOOKUP([@store_id], raw_stores!A:F, 2, FALSE)

Para departamentos:

=XLOOKUP([@dept], raw_departments!A:A, raw_departments!B:B, "No Match")

2.3 Validación del merge

Crea una hoja JOIN_DOC con:

validación resultado esperado
Conteo de No Match en tiendas 0
Conteo de No Match en departamentos Revisar: hay un dept intencionalmente inexistente
Duplicados en clean_sales Identificar y decidir si se eliminan
Total ventas antes vs. después de limpieza Debe reconciliar o explicar diferencias

No elimines un registro sin documentar la decisión.

Parte 3 — KPIs principales (20 min)

Crea tres hojas:

kpi_efficiency
kpi_growth
kpi_share

KPI 1 — Eficiencia: ventas por m²

Definición:

Sales_per_m2 = SUM(weekly_sales) / size_m2

Pivot sugerida:

Área Campo
Rows store_name
Columns store_type
Values SUM weekly_sales
Helper dividir por size_m2

Interpretación: una tienda pequeña puede ser más eficiente que una tienda grande si genera más ventas por metro cuadrado.

KPI 2 — Crecimiento semanal

Definición:

Weekly_growth_pct = (sales_current_week - sales_previous_week) / sales_previous_week

Pasos sugeridos:

  1. Crea una Pivot por week_key con suma de ventas.
  2. Ordena semanas ascendentemente.
  3. Agrega una columna de crecimiento.
  4. Evita división por cero o semanas vacías.

KPI 3 — Participación por departamento

Definición:

Dept_share_pct = department_sales / total_sales

Pivot sugerida:

Área Campo
Rows dept_name
Values SUM weekly_sales
Show as % of grand total

Si la herramienta no permite porcentaje del total, calcula fuera de la Pivot:

=B2/SUM($B$2:$B$10)

Validación

Cierre y validación de conocimiento (6 min)

  1. ¿Por qué no se debe limpiar directamente sobre raw_sales?
  2. ¿Qué problema resuelve TRIM()?
  3. ¿Por qué se crea una columna week_key?
  4. ¿Qué significa un resultado No Match en un lookup?
  5. ¿Por qué Sales_per_m2 puede cambiar la lectura frente a ventas totales?
  6. ¿Qué validación harías antes de confiar en una Pivot?
  7. ¿Qué diferencia hay entre una cifra y una iniciativa en C→F→I?

Respuestas o criterios esperados

  1. Para preservar la fuente original y poder auditar cambios.
  2. Elimina espacios extra que afectan llaves, categorías y búsquedas exactas.
  3. Permite agrupar semanas de forma consistente.
  4. La llave no existe en la dimensión o está mal estandarizada.
  5. Normaliza ventas por tamaño de tienda; permite comparar eficiencia.
  6. Reconciliar totales, revisar matches, fechas válidas y duplicados.
  7. La cifra describe evidencia; la iniciativa propone una acción o análisis siguiente.

Takeaways

  • La práctica profesional exige conservar datos crudos y trabajar en copias.
  • La limpieza debe ser documentada, no solo aplicada.
  • Los lookups conectan hechos con dimensiones y permiten análisis más ricos.
  • Un KPI debe tener definición explícita y unidad de medida clara.
  • Las tablas dinámicas aceleran el resumen, pero no reemplazan la validación.
  • El dashboard debe comunicar pocos elementos bien seleccionados.
  • C→F→I ayuda a pasar de datos a decisión.
