## Objetivos de Aprendizaje
- Aplicar los **comandos Git de trabajo remoto**: `fetch`, `merge`, `pull`, `push`
- Configurar Git con `config` y gestionar **ramas** con `branch` y `checkout`
- Subir un repositorio completo a **GitHub** de forma autónoma
- Usar **`.gitignore`** para excluir archivos innecesarios
- Establecer el hábito de usar Git en todos los proyectos del bootcamp

## Resumen de Secciones y Contenido
1. **Pregunta Guía** – ¿Cómo colaborar con otros y mantener tu código seguro en la nube?
2. **Repaso** – comandos básicos de Sprint 2 class3
3. **Flujo completo con GitHub** – diagrama: Working Dir → Staging → Local Repo ↔ Remote Repo
4. **Comandos remotos** – `remote add origin`, `push -u origin main`, `push`, `fetch`, `pull`, `clone`
5. **Ramas y navegación** – `branch`, `checkout`, `merge`, `tag`
6. **`.gitignore`** – ejemplo completo para DS: `.venv/`, `*.pyc`, `.env`, `data/raw/`, `.DS_Store`
7. **Demo en vivo con voluntario** – crear repo en GitHub, conectar, push, ver historial
8. **Breakout Rooms** – crear repo `sprint3-pandas-analisis`, estructura, commits, push
9. **Tips** – pull before push, branches for features, meaningful commit messages
