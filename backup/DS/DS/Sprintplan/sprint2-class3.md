## Objetivos de Aprendizaje
- Comprender qué es el **control de versiones** y por qué es fundamental para DA/DS
- Conocer los **conceptos clave de Git**: repositorio, commit, rama, tag, remoto
- Aplicar los **comandos básicos**: `init`, `status`, `add`, `commit`
- Entender la diferencia entre **Git** (herramienta) y **GitHub** (servicio)
- Subir tu primer repositorio a **GitHub**

## Resumen de Secciones y Contenido
1. **Pregunta Guía** – ¿Cómo colaboran los equipos de datos y cómo protegen su trabajo?
2. **El problema sin control de versiones** – `analisis_final_ESTE_SI_de_verdad.py` analogy
3. **Qué es control de versiones** – Git vs SVN vs Mercurial; por qué Git ganó
4. **Por qué es fundamental para DA/DS** – reproducibilidad, colaboración, portafolio, entrevistas
5. **Conceptos clave** – glosario: repo, commit, branch, remote, origin, tag; 3 estados de un archivo
6. **Comandos básicos** – `git config`, `git init`, `git status`, `git add`, `git commit`, `git log`; demo paso a paso
7. **GitHub** – Git vs GitHub tabla; pasos para subir: `remote add origin`, `push -u origin main`
8. **Breakout Rooms** – crear repo, 3 commits, push a GitHub
9. **Kahoot** – 6 preguntas
