# Sprint 7 – Notebook 1: Línea de Comandos y Git/GitHub

## Archivo (to be created)
`DS/Sprint 7/DS_Sprint7_class1.ipynb`

## Sprint
**SPRINT 7 – Herramientas de Desarrollo de Software**

## Tema
Línea de Comandos + Git y GitHub

## Objetivos de Aprendizaje
- Navegar el sistema de archivos y ejecutar comandos básicos en la **terminal (CLI)**
- Comprender qué es **Git** y por qué es esencial en Data Science
- Aplicar el flujo básico de Git: `init`, `add`, `commit`, `push`, `pull`
- Trabajar con **ramas (branches)** y hacer **pull requests**
- Publicar un repositorio en **GitHub** con buenas prácticas

## Resumen de Secciones y Contenido
1. **Pregunta Guía** – ¿Cómo colaboran los equipos de Data Science en proyectos reales?
2. **Línea de comandos** – `pwd`, `ls`/`dir`, `cd`, `mkdir`, `cp`, `mv`, `rm`; rutas absolutas vs relativas; por qué los DS necesitan la terminal
3. **Git conceptos** – qué es control de versiones; repositorio, commit, staging area; analogía con "guardar versiones de un documento"
4. **Flujo básico de Git** – `git init`, `git add`, `git commit -m`, `git log`, `git status`, `git diff`
5. **Ramas y colaboración** – `git branch`, `git checkout`, `git merge`; qué es un pull request; flujo de trabajo en equipo
6. **GitHub** – crear repositorio, `git remote add origin`, `git push`, `git pull`, `git clone`; `.gitignore`
7. **Actividad práctica** – students create a repo, make commits, and push to GitHub; spinner to pick presenter
8. **Tips y buenas prácticas** – commit often, write meaningful messages, never commit secrets, use `.gitignore`
9. **Cierre y Kahoot**

## Requerimientos
- Git installed locally
- GitHub account
- Terminal (bash/zsh/PowerShell)

`git`, `github.com`
