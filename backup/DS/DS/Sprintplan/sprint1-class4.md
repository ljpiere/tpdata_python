## Objetivos de Aprendizaje
- Comprender qué es el **debugging** y cuándo aplicarlo

- Aplicar estrategias sistemáticas para encontrar y corregir errores

- Instalar y usar **extensiones útiles** para Data Science

## Resumen de Secciones y Contenido
1. **Pregunta Guía** – ¿Cómo encontrar y corregir errores de forma sistemática?
2. **Qué es debugging** – origen (Grace Hopper, 1947); cuándo aplicarlo
3. **Estrategias** – leer el error, print(), reproducir el bug, buscar en internet, debugger, LLMs
4. **Debugger de VSCode** – breakpoints (simple, condicional, hit count, logpoint); F10/F11/F5; panel Variables; panel Watch
5. **IntelliSense** – tabla de símbolos (🟣 método, 🔵 propiedad, 🟡 variable, etc.); linting (líneas onduladas)
6. **Extensiones** – tabla esenciales (Python, Pylance, Jupyter, GitLens, Git Graph) + productividad (Better Comments, TODO Tree, Project Manager, Prettier)
7. **Tipos de errores** – tabla completa: SyntaxError, NameError, TypeError, ValueError, KeyError, IndexError, AttributeError, ImportError, FileNotFoundError, ZeroDivisionError
8. **Breakout Rooms** – debug a buggy script with 3 bugs
9. **Kahoot** – 6 preguntas
